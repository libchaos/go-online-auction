## 仓库分析：go-online-auction 架构与启动
- 项目：Go + React 实时在线拍卖系统。后端 Go 1.25.5，六边形架构 + CQRS + 事件驱动 + DDD，Uber Fx 依赖注入，Cobra CLI 入口。
- 消息中间件实为 **NATS JetStream**（非 Redis，README 正文部分描述有误，Tech Stack 表格正确）。命令流 StreamCommands + BidProcessor 消费者（去重/重试/DLQ），事件通过 dispatcher 推送到 WebSocket Hub。
- 实际后端端口为 **9000**（.env.example: HTTP_SERVER_PORT=9000），README 正文里的 8080 是笔误。前端 Vite :5173，DB :5432，NATS :4222/8222。
- 分层：internal/modules/auction/{domain,application(command/query),ports,infra(persistence/messaging/websocket/http-chi)}；shared 模块含 config/database/httpserver/logger/nats。
- CLI 命令：all / auction / websocket / db:migrate。Makefile：make run / run-auction / run-websocket / migrate / test / lint。
- 启动顺序：docker-compose up -d → cp .env.example .env → make migrate → make run → 前端 frontend-demo: npm install && npm run dev。
- 并发控制：领域校验 + SELECT FOR UPDATE NOWAIT + version 乐观锁 + DB 触发器兜底。

## 设计文档：策略模式实现多交易模式
- 产出：`docs/trading-mode-strategy-design.md`。
- 切入点：现有 `AuctionModel.PlaceBid`（`domain/model/auction_model.go:139`）硬编码英式逻辑（`ErrBidMustExceedHighest`），异步侧 `BidProcessor.ProcessBid`（`infra/messaging/bid_processor.go:144`）调用之。
- 方案：新增 `domain/enum/trading_mode_enum.go`（english/dutch/sealed_bid/vickrey/fixed_price）+ `domain/strategy` 包（TradingStrategy 接口 + 5 实现 + Resolver）。模型加 tradingMode/startingPrice/priceStep/reservePrice/currentPrice/winner 字段；PlaceBid 委托策略，Close(bids) 用策略算 winner。
- 装配：Fx `fx.Group:"trading_strategy"` 收集策略切片 → NewResolver；BidProcessor 注入 Resolver 处理 ShouldCloseOnAccept（荷兰式/一口价出价即结束）；IsPermanentBidError 扩展新错误；AuctionEndedEvent 富化 winner。
- 迁移：auctions 表加 trading_mode/starting_price/price_step/reserve_price/current_price/winner_* 列，存量默认 english。分阶段实施清单见文档。
