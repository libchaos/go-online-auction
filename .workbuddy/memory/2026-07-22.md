## 修复 fx 启动 DI 断链（缺 sqlcgen.DBTX provider）
- 现象：`go run ./main.go all` 在组装 `auction.RegisterAuctionRoutes` 时报 `failed to build ports.DepositRepository: missing type: sqlcgen.DBTX`。
- 根因：deposit、payment 两个模块的 `Module` 漏了 `fx.Provide(func(pool *pgxpool.Pool) sqlcgen.DBTX { return pool })`。users/listing/auction/notification 都有此 provider，deposit/payment 没有；ledger 的 UoW factory 直接吃 `*pgxpool.Pool`、repository 在 factory 内创建，故 ledger 不需要该 provider。
- 修复：在 `internal/modules/deposit/module.go` 与 `internal/modules/payment/module.go` 各加该 provider 及 import（`github.com/jackc/pgx/v5/pgxpool` + 各自 `infra/sqlcgen`）。
- 验证：`go build ./...`=0；`golangci-lint`(deposit+payment)=0 issues。完整进程启动需用户环境 Postgres+NATS（本沙箱无，未实跑）。
- 约定（记入 MEMORY）：凡模块内有 sqlcgen 生成的 repository 直接吃 `sqlcgen.DBTX`，其 `Module` 必须提供 `fx.Provide(func(pool *pgxpool.Pool) sqlcgen.DBTX { return pool })`，否则全图 `all` 启动时会在首个依赖该模块的 INVOKE 处断链。

## 修复 fx 启动 DI 断链（deposit Hub 缺 websocket.EventConsumer 绑定）
- 现象：`go run ./main.go all` 在 `deposit.RegisterDepositWebsocketRoutes` 报 `failed to build *websocket.Hub: missing type: websocket.EventConsumer`。
- 根因：deposit websocket `Hub`（`infra/websocket/deposit_hub.go`）依赖本地 `websocket.EventConsumer` 接口；其实现 `messaging.NewJetStreamDepositEventConsumer` 仅以 `*JetStreamDepositEventConsumer` 提供，未 `fx.As(new(websocket.EventConsumer))`，导致 fx 找不到该接口绑定。同模式参考：auction=`fx.As(new(messaging.EventConsumer))`、notification=`fx.As(new(sse.EventConsumer))`。
- 修复：`internal/modules/deposit/module.go` 将 `fx.Provide(messaging.NewJetStreamDepositEventConsumer)` 改为 `fx.Annotate(messaging.NewJetStreamDepositEventConsumer, fx.As(new(websocket.EventConsumer)))`。
- 验证：`go build ./...`=0（BUILD_OK）。payment 模块无 websocket Hub，无同类缺口。

## 修复 fx 启动 DI 断链（notification 漏提供 config.Email）
- 现象：`go run ./main.go all` 在 `notification.RegisterNotificationEmailConsumer` 报 `failed to build ports.EmailPort: missing type: config.Email`（经 `email.NewSMTPEmailAdapter(cfg config.Email)`）。
- 根因：`config.Email` 是 `config.Config` 的 squash 子结构，全局只提供 `config.Config`；各模块需自行 `fx.Provide(func(cfg config.Config) X {...})` 抽取子配置。notification 抽了 `notificationoutbox.Config` 却漏抽 `config.Email`，导致 `NewSMTPEmailAdapter` 拿不到 `config.Email`。payment 模块已有 `fx.Provide(func(cfg config.Config) config.Alipay {...})` 同模式。
- 修复：`internal/modules/notification/module.go` 补 `fx.Provide(func(cfg config.Config) config.Email { return cfg.Email })`（放在 outbox config provider 旁）。
- 验证：`go build ./...`=0（BUILD_OK）。
- 通用规律（记入 MEMORY Fx 约定）：fx 图只提供根 `config.Config`；任何需要 squash 子配置（`config.Email`/`config.Alipay`/`config.DB`/各 `outbox.Config`/`scheduler.Config` 等）的构造器，模块必须显式 `fx.Provide(func(cfg config.Config) X { return cfg.X })` 抽取，否则按子类型缺失断链。
