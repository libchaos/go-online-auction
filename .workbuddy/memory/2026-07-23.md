### 关键坑：响应双层 `data` 包络
- 后端 `internal/shared/sdk/http/response.NewEnvelope(data)` 把所有成功响应包成 `{ "data": <payload> }`。
- openapi.yaml 的 200 schema 又是 `{ data: <XxxResponse> }`，所以响应体是**双层 data**：`{ data: { data: <payload> } }`。
- 前端取数必须 `const { data } = await client.GET(...); const payload = data?.data;`（外层 data 是 openapi-fetch 的，内层 data 是后端 Envelope 的）。已修复 auctions/auctions.$id/deposits/login/me/notifications/watchlist 共 7 个文件的取数。
- `GET /api/v1/deposits` **无查询参数**（openapi 未定义 limit/offset），已去掉 deposits 列表的请求查询参数。
